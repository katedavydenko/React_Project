4. 1. Підняття стану — це перенесення стану з дочірнього компонента до найближчого спільного батьківського. Для `SearchBar` це потрібно, щоб значення пошуку зберігалося в `App` і могло використовуватися для фільтрації постів.

2. Оновлення стану через `setState` (або `setSomething`) відбувається асинхронно — React може об’єднувати кілька оновлень для оптимізації. Тому нове значення стану не змінюється миттєво після виклику функції оновлення.

3. Тому що стан у React має бути незмінним (immutable). Замість зміни оригінального масиву ми створюємо новий, щоб уникнути побічних ефектів і зберегти початкові дані.

4. Керовані компоненти зберігають значення поля вводу у стані React, що дає повний контроль над даними, дозволяє легко виконувати фільтрацію, валідацію та синхронізацію з іншими компонентами.
